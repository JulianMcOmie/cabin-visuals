MIDI Vim - Architecture

A modal editing language for the piano roll: a caret on the (beat × visible-row) grid, vim's count-motion-operator grammar aimed at notes, and view commands that drive the row model - which is exactly why the row-hiding work is this feature's substrate, not its sibling.

2 · The starter keymap (the contract to argue about BEFORE code)

KeysAction
h l / j k · countsCaret by grid step / visible row; 4l = a beat at 1/4 grid. Caret is visible as a cell outline; independent of selection.
i / aInsert a note at the caret (length = quantize; velocity = last-used) / same but advance the caret one grid step after - rapid step entry is i l i l… or just a a a.
x · dd-style dwDelete note under caret · delete to next note.
v + motionsVisual mode: extends a note selection (the EXISTING selection set - vim drives the same selection mouse gestures use).
y pYank selection / put at caret (reuses the note-clipboard the editor already has).
< > · J K on selectionNudge selected notes by grid step · transpose by visible row.
u / Ctrl-rExisting undo/redo, untouched.

3 · Decisions & non-goals

DecisionWhy
Opt-in mode, per the piano roll onlyA toolbar toggle (and one hotkey) turns vim mode on; off = today's editor untouched. The timeline never gets modes.
Vim drives EXISTING actionsEvery operator maps to the same note-mutation paths the mouse uses (commit-per-gesture undo, same clipboard, same selection set) - the keymap is a new INPUT layer, not a new EDITING engine. This is the whole mess-avoidance strategy.
Row model first - hard dependencyMotions over hidden/filtered rows are the point ("fixed long distances", scale-degree movement). Building vim on raw pitches first would mean rebuilding it after.
Keymap doc before keystroke one§2 is a proposal for Julia (and the promised "midi vim discussion"); the grammar is cheap to argue on paper and expensive to rewire after muscle memory forms.
Non-goalsEx commands / command line · macros (record/replay - tempting, later) · registers beyond one · vim in the timeline or track editor · remapping UI (a constants file is enough for two users).
The one-sentence mental model: a caret on the row model's grid, vim's count-operator-motion grammar translated onto the editor's existing selection/clipboard/undo machinery, and view filters bound to keys - a new input layer over old truth, never a second editor.
