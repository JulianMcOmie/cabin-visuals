Incremental Resolve — Architecture

Stop re-resolving the whole project on every edit — but only when a real project proves it's slow, and only by the cheapest honest trick: the store is immutable, so "did this track change?" is a reference comparison. Memoize per-track compilation; keep global assembly a full, cheap pass.

1 · Split resolve at its natural joint

2 · Why reference-keyed memoization and nothing fancier

AlternativeVerdict
Dependency-graph invalidation (track → dependents)Rejected. Routings/tags/solo make "dependents" cross-cutting; modeling them buys correctness risk to optimize the part that's already cheap (assembly). This is the fiddliness the backlog note warned about.
Reference-memoized per-track compile + full assemblyChosen. The store already guarantees changed track ⇒ new reference (HistoryStore and autosave both rely on it). The cache is a WeakMap from track ref → compiled output: no eviction policy, no staleness class of bugs — an unchanged ref CANNOT have changed content.
Web-worker resolveDifferent medicine (moves cost off-thread instead of removing it); noted as the escalation if profiling shows assembly itself hurting at extreme scale.

3 · Decisions & non-goals

DecisionWhy
Measure first — a numeric gate, not vibesThe feature starts with a resolve-time probe on real projects. Gate: proceed only if debounced resolve exceeds ~8ms at realistic size (or Julia feels edit lag attributable to it). Below that, close the item as "not yet".
Output-equality is the acceptance testIncremental resolve must produce a graph deep-equal to the full resolve on a corpus of real documents — a property test run in CI-by-hand, not a hope.
beatsPerBar changes flush the cacheIt's baked into flattening; it's one global key beside the WeakMap.
syncParams fast-path unchangedSlider-drag reactivity already bypasses resolve; this work only shrinks the debounced structural pass.
Non-goalsIncremental ASSEMBLY · worker offload · caching across project loads · any change observable in the graph's shape.
The one-sentence mental model: the store's immutability already computed the dirty bits — memoize the per-track half of resolve on track references, keep the cross-cutting half a dumb full pass, and gate the whole effort behind a stopwatch reading that says it's actually needed.
