Copy-Paste Automation — Implementation Plan

Phase 0 is one question to Tyler; the code is a branch in the existing paste handler plus a small re-target helper. Assumes the "duplicate a lane onto another instrument" spec from the architecture doc.

B · Phase-by-phase

Phase 0 — Confirm the spec ★ Tyler

  • One question: "copy the whole lane onto another instrument, or copy selected keyframes between two existing lanes?" Whole-lane → this plan. Keyframes → close this and file a piano-roll item instead.

Phase 1 — The re-target helper ProjectStore

  • new action pasteAutomationTrack(sourceTrack: Track, newParentId: string): 'ok' | 'duplicate' | 'no-param': clone the lane (fresh ids via cloneTrack), set parentId = newParentId, append to the parent's childIds; if the parent instrument has targetParam → keep; else keep the key but return 'no-param' (the lane carries its keyframes and shows the param-not-found state); if a sibling automation lane already targets that param → return 'duplicate', no mutation.
  • Values need no conversion: keyframes are pitch-encoded; decode happens against the CURRENT parent's param bounds at resolve time (proportional mapping is emergent).

Phase 2 — The paste branch useTrackGestures

  • In the Ctrl+V clip.kind === 'track' path: if clip.track.type === 'automation' — require a selected track whose instrument exists; call the helper; select the new lane on 'ok'; toast on 'duplicate' / 'no-param' / no-selection (sonner is already a dependency).
  • Non-automation tracks keep today's root-paste behavior byte-for-byte.

Phase 3 — Verify

  • Cube A with a Base X lane (0→10 sweep) → copy → select Cube B → paste: B gains the lane, sweep plays on B, A unchanged; undo removes only B's lane.
  • Paste onto a track already automating Base X → toast, no change. Paste onto Circle (has the param) → works. Paste onto an instrument lacking the param → lane lands in not-found state, re-aim via the param picker.
  • Persistence round-trip: save/reload keeps the pasted lane's parent linkage.

D · Risks & open questions

  • ★ open The spec. Do not build before Tyler answers — the alternate reading is a different, cheaper feature.
  • risk Proportional mapping surprises: a hue sweep (0–360) pasted onto a size param (0.2–4) is "correct" proportionally but may feel wrong; the not-found/param-picker affordance doubles as the escape hatch (re-aim + the lane keeps its shape).
  • note Depends lightly on the interp-discoverability phase-0 fallback (the param-not-found lane state) — land that first or inline the same fallback here.
