3D Model Import - Implementation Plan

Four phases, three of them transcriptions of shipped audio code: bucket + source cache, catalog + persistence ride-along, the Model instrument, upload UI. The only genuinely new engineering is glTF normalization and the parse cache.

B · Phase-by-phase

Phase 1 - Bytes plumbing persistence + core

  • new persistence/modelStorage.ts: transcribe audioStorage (upload/getUrl/delete) against a project-models bucket; same RLS policy shape, applied via the Drizzle RLS setup like the audio bucket was.
  • new core/visual/modelSource.ts: transcribe audioSource - saveModel(File)→ref with the no-project session fallback, getModelUrl(ref) with the object-URL cache, removeModel.
  • new core/visual/modelCache.ts: getModelScene(ref): Promise<Group> - fetch → GLTFLoader.parse (cached by ref, failures uncached) → NORMALIZE: recenter to bbox center, uniform scale into a 1.6 box; return a template the instrument CLONES per use (two tracks wearing one ref must not share mutable transforms).
  • Verify: node-side none (browser APIs); in-app: upload → ref resolves → parse logs dimensions → normalization puts any test model in a ~1.6 box.

Phase 2 - Catalog + persistence stores · serialize

  • new store/ModelStore.ts: modelAssets: Record<ref, {fileName: string}> + add/remove - AudioStore's twin.
  • edit persistence/serialize.ts: doc gains modelAssets beside audioClips (pick + hydrate with ?? {}); persistence/types.ts optional field. No version bump - absent reads as empty.
  • Verify: save/reload round-trips the catalog; an old document loads with an empty catalog silently.

Phase 3 - The instrument instruments/Model.tsx

  • Def: params (baseSize/baseX/Y/Z/spinSpeed sliders + modelRef as a string param), the standard ports, localTransform copied from Cube's (breathe × energy, opt-in spin).
  • Component: useInstrumentFrame for tint/emissive (hue-tint params lerp cached material colors - clone materials once per instance); suspense-free async load: placeholder wireframe box until getModelScene resolves, then swap the cloned scene in.
  • Registry + library tile ("Model" under Object). Selecting a Model track shows a "Choose model…" row in the TrackEditor: existing assets from the catalog + an Upload button (file input → saveModel → catalog add → setTrackStringParam('modelRef', ref)).
  • Verify: upload a GLB → renders normalized; notes pulse it; a clone effect multiplies it; export includes it (nothing special - it's scene furniture); missing ref (asset deleted) → placeholder + no crash; two tracks, one ref → independent transforms.

Phase 4 - Guardrails

  • 25MB pre-upload check with a readable toast; accept .glb/.gltf only (.gltf with external buffers is rejected with "use GLB" guidance - the single-file constraint keeps the bucket model trivial).
  • Poly sanity: log triangle count on parse; above ~500k warn in the picker row ("heavy model - may drop frames"). No hard block.
  • Asset delete from the picker: removes catalog entry + bucket object (fire-and-forget like audio); tracks wearing it fall back to placeholder.

D · Risks & open questions

  • risk Pause invariant vs glTF content: some GLBs carry animations that drei-style helpers auto-play - the loader path must NOT start mixers. Static scene extraction only; the canary will catch violations instantly (its job).
  • risk Texture memory on meme-sized PNGs inside GLBs - the 25MB cap bounds it roughly; revisit with real usage.
  • risk Material cloning double-count (tint needs clones; clone EFFECTS clone again) - the tint clones at instrument mount, effects clone via CloneSpec instancing - confirm the interaction on one test scene early in phase 3.
  • open Default hue-tint amount 0 (model's own colors) - assumed; confirm with Julia.
  • note Beat-sampled glTF animation is the killer follow-up ("the face lip-syncs") - the modelCache API should return clips metadata now (unused) so the follow-up doesn't re-parse.
