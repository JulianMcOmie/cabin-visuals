B · Radial Copy - the def

// effects/clones/radialCopy.ts - pure CloneSpec producer, per the house pattern
export const radialCopyPlugin: VisualEffect = {
  id: 'radialCopy',
  name: 'Radial Copy',
  category: 'clone',
  settings: [
    { key: 'count',        label: 'Count',         min: 2, max: 16,  step: 1,    default: 6 },
    { key: 'radius',       label: 'Radius',        min: 0, max: 8,   step: 0.05, default: 2 },   // zero-safe
    { key: 'angle',        label: 'Angle',         min: 0, max: 360, step: 1,    default: 0 },   // phase-complete
    { key: 'scaleFalloff', label: 'Scale Falloff', min: 0, max: 1,   step: 0.01, default: 0 },
  ],
  clones: (s) => Array.from({ length: s.count }, (_, i) => {
    const a = ((s.angle + (360 / s.count) * i) * Math.PI) / 180
    return {
      position: [Math.cos(a) * s.radius, Math.sin(a) * s.radius, 0],
      scale: 1 - s.scaleFalloff * (s.radius > 0 ? 1 : 0) * (i / Math.max(1, s.count - 1)),
    }
  }),
}

Whether the source stays visible at the center follows the sibling clones' convention - match echo/tile, don't invent.

C · Phase-by-phase

Phase 0 - Gate dependency

  • Effect-param automation merged and its keystone canary green. Building these first would just add params nobody can animate.

Phase 1 - Radial Copy effects/clones/radialCopy.ts

  • New file per the def above + registry entry - it appears in the library and the picker menu automatically.
  • Verify statically first: sliders at rest - count/radius/angle/falloff behave; radius 0 collapses to the source (zero-safe check); angle 0 ≡ angle 360 (phase-complete check).

Phase 2 - Upgrades linearDuplicate · rotationalSymmetry

  • linearDuplicate: spacing min → 0 (zero-safe), add drift (0–1 of one spacing unit, phase-complete along the line). Defaults preserve current output exactly - existing projects render identically (compare a saved scene before/after).
  • rotationalSymmetry: expose angleOffset (0–360) if internal; bounds audit for the rest.

Phase 3 - The standard effects/types.ts

  • Three-line doc comment on the settings type: motion params are continuous, zero-safe, phase-complete where angular; counts are steps and stay coarse. Cite Radial Copy as the reference implementation.

Phase 4 - The demo verify (this feature's reason to exist)

  • Cube + Radial Copy; lanes on radius (0→5, 4 bars, smooth-step) and angle (0→360, looped block once loops land - linear until then). Play: the ring breathes and spins. Park anywhere: frozen exactly (canary). Export: file ≡ editor.
  • Falloff lane + radius lane together - two lanes on one instance, independent (the instanceId addressing test from the automation pair, exercised for real).

E · Risks & notes

  • risk Upgrades changing existing scenes - the phase-2 before/after render comparison is the gate; defaults must reproduce today's output bit-for-bit.
  • risk Count × clones per frame: 16 clones on a heavy instrument is 16× draw - fine for primitives; note in the plugin's About text that count multiplies cost.
  • note The angle lane's perpetual-motion trick wants looped automation blocks - cross-reference the loop-blocks pair; works with a manual 0→360 ramp per section meanwhile.
