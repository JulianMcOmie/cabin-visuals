B · The expansion (the engine truth)

// core/visual/resolve.ts - inside flattenBlocks, per block
const beatsPerLoop = (block.loop && block.loopBars ? block.loopBars : block.durationBars) * beatsPerBar
const blockBeats = block.durationBars * beatsPerBar
const repeats = block.loop && block.loopBars ? Math.ceil(blockBeats / beatsPerLoop) : 1
for (let k = 0; k < repeats; k++) {
  const offset = k * beatsPerLoop
  for (const n of block.notes) {
    const start = n.startBeat + offset
    if (start >= blockBeats) continue                       // partial repeat clips
    emit({ ...n, startBeat: start,
           durationBeats: Math.min(n.durationBeats, blockBeats - start) })
  }
}

Pure data→data at resolve; per-frame cost unchanged. Notes authored past loopBars (possible via older edits) are ignored while looping - the pattern window is bars 0..loopBars.

C · Phase-by-phase

Phase 1 - Data + expansion ★ keystone types · resolve

  • edit types.ts: loopBars?: number beside the existing loop. Optional → no document upgrade.
  • edit core/visual/resolve.ts as above.
  • Verify BEFORE any UI (set flags via a temp store call): 2-bar pattern, 5-bar block, loopBars 2 → notes at bars 0–2, 2–4, 4–5(clipped); scrub into repeat 3 ≡ play into repeat 3; canary green.

Phase 2 - The gesture useTrackGestures

  • In resizing-right: capture patternBars = block.loop ? block.loopBars : durationBars at drag start. During drag: newDuration > patternBars → write { durationBars, loop: true, loopBars: patternBars }; newDuration <= patternBars → write { durationBars, loop: false, loopBars: undefined }.
  • Quarter-beat snapping applies unchanged; min duration unchanged; resizing-LEFT on a looping block keeps loopBars and shifts the pattern window with the counter-shifted notes (existing left-resize note math already does the counter-shift).
  • Verify: drag out → loop engages exactly past the pattern edge; drag back in → block returns to a plain block byte-identically (flags cleared).

Phase 3 - Previews Block.tsx · piano roll

  • Block.tsx: NotePreview tiles the pattern dashes at 55% opacity for repeats; a 1px dashed vertical line at each loop boundary.
  • Piano roll: repeats render as dimmed, non-interactive notes past the pattern window; the block-window shading marks the pattern region distinctly. Gestures refuse targets past loopBars (rowModel/gesture guard).
  • Verify: visual boundaries align with the ruler at loopBars multiples; ghosts update live while editing pattern notes.
