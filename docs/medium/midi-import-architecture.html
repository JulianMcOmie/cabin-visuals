MIDI Import - Architecture

A .mid file becomes tracks of blocks of notes - entirely in the beat domain, so the file's tempo map is irrelevant by construction. Import keeps full pitch fidelity; narrow-range instruments and the piano roll's future row tools handle display, never the data.

1 · The pipeline - ticks to beats, never seconds

2 · The pitch-range question, answered as decided

Julia's original question: instruments with narrow editors (7 rows) vs files spanning the pitch spectrum. The decision recorded in the backlog stands: import faithfully - the data keeps full pitch; the view reconciles.

LayerBehavior with out-of-range pitches
DocumentEvery note imports verbatim. Re-aiming the track at a wider instrument later loses nothing.
InstrumentAlready ignores pitches it doesn't map - importing a full-range file into a 7-lane instrument simply plays the 7 lanes it understands.
Piano rollShows everything today; the row-hiding/scaling item (its own doc pair) adds "show only mapped rows" so narrow instruments read cleanly.
Import summaryThe one nicety: after import, a toast counts what landed - "142 notes · 38 outside Cube's range" - so silence isn't mysterious.

3 · Decisions & non-goals

DecisionWhy
One new track per non-empty MIDI trackMatches the file's structure; named from the MIDI track name; default instrument (Cube) with the track selected so swapping is one click. Importing INTO an existing track is the drag-onto-lane variant, later.
One block per track, spanning first→last noteBlocks are the editing unit; a single block rounded up to whole bars is the least-surprise landing shape. Split-at-silence is a refinement, not v1.
Velocity imports 0–127 as-isInstruments already tolerate both 0–1 and 0–127 scales.
Grow the project if the file overrunsIf the last note ends past totalBars, bump totalBars (it's one store field, undoable) rather than truncating someone's song.
Non-goalsTempo/time-sig import · CC/pitch-bend → automation (tempting, but it needs the richer automation targets first - noted as follow-up) · MIDI export.
The one-sentence mental model: ticks over PPQ are already beats, and beats are already the document's native time - so import is a parser walk that mints tracks/blocks/notes verbatim, tells you what a narrow instrument will ignore, and leaves display problems to the view layer where they belong.
