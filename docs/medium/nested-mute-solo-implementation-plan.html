B · The derived-state helper

// core/trackTypes.ts (or a sibling) — pure, resolve-time, no per-frame work
function effectiveMuteSolo(tracks, trackId): { muted: boolean; soloed: boolean } {
  let muted = false, soloed = false
  for (let t = tracks[trackId]; t; t = t.parentId ? tracks[t.parentId] : undefined) {
    muted ||= t.muted          // Q1(b): any ancestor's mute silences you
    soloed ||= t.solo          // Q2(b): any ancestor's solo includes you
  }
  return { muted, soloed }
}
// anySolo remains "any OBJECT track effectively soloed";
// objectOff = eff.muted || (anySolo && !eff.soloed)   — same formula, effective inputs

O(depth) per track, computed once per resolve. If Tyler's answers differ from the recommendation, only this helper changes — that's the point of it.

Phase 3 — Button states Track.tsx

  • Three visual states per button: off · own (solid amber/green, as today) · inherited (dim amber/green, not clickable-off from the child — clicking sets the CHILD's own flag, Logic-style).
  • The paint-drag gesture keeps painting OWN state only; inherited rendering is display-only.
  • Verify: dim state appears on all descendants when a parent mutes; a dim child clicked becomes solid (own mute layered under parent's).
