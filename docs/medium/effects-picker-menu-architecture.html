Effects Picker Menu - Architecture

Logic's plugin-slot gesture for the effects tab: an "+ Add effect" slot opens a nested category menu (Transform ▸ / Clone ▸ / Shader ▸) fed by the same registry the drag-in library uses. Drag-in stays; this is the second door to the same room.

2 · Decisions & non-goals

DecisionWhy
The slot lives at the END of the instance listLogic's mental model: existing effects stack top-to-bottom, the empty slot below invites the next one. Clicking an existing instance keeps opening its settings, unchanged.
Generated from the registry, grouped by categoryOne source of truth. Category display names/order defined once (Transform · Clone · Shader).
Duplicates allowedaddEffect already permits two of the same plugin (two offsets compose); the menu doesn't grey out added items - unlike the automation menu, where one-lane-per-param is the rule.
Same menu on right-click of the effects listFree once the component exists; both roads call the same addEffect(trackId, pluginId).
Non-goalsSearch/filter in the menu (12 items don't need it) · replacing drag-in · effect reordering (separate concern; note if asked) · favorites.
The one-sentence mental model: the registry already knows the catalog and the timeline already grew a viewport-safe nested menu - this feature is one "+ Add effect" slot that marries them and calls the store action drag-in has always called.
