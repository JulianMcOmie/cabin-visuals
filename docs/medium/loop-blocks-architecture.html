Loop Blocks — Architecture

Drag a MIDI block's right edge past its pattern and the pattern repeats — Ableton's gesture, landed on the existing (unused) loop flag. Expansion happens once at resolve time; playback, scrubbing, and export never know loops exist.

DecisionWhy
loopBars joins loop on BlockThe flag alone can't say what repeats. Optional field → old documents read as undefined → no loop → no schema bump, no upgrade step.
Resize-past-end IS the loop switchJulia's spec verbatim; no new buttons. Plain resize inside the pattern stays plain resize.
Pattern-only editing, ghost repeatsOne source of truth for notes; the piano roll shows repeats dimmed and read-only past loopBars (editing a ghost jumps you to its pattern note — later nicety, not v1).
Partial repeats clipA 2-bar pattern in a 5-bar block plays 2+2+1; clipping at the block end is what flattenNotes already does with block windows.
Non-goalsLoop offset/phase (start mid-pattern) · per-repeat variation · looping AUDIO blocks (different mechanics entirely) · transport loop region (that's the export-ranges companion, not this).
The one-sentence mental model: the right edge past the pattern locks the pattern length into loopBars, resolve tiles the notes to fill the block, and everything downstream — engine, scrub, export — keeps seeing ordinary notes, which is why the whole feature is one gesture, one preview treatment, and one expansion loop.
