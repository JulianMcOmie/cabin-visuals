Tag Automation - Architecture

"Tagging is so good for this" - one automation lane drives one param key across EVERY instrument carrying a tag. The lane goes normalized (0–1) and each instrument maps it into its own bounds, which is what makes cross-instrument fan-out coherent instead of accidental.

1 · The fan-out - one lane, N instruments, per-instrument bounds

2 · Why normalized lanes

AlternativeVerdict
Absolute values (lane in the param's units)Whose units? #drums instruments can disagree about bounds for the same key, and unrelated keys can't share a lane at all. Dead end.
Normalized 0–1, mapped per instrumentChosen. The lane says "how far through YOUR range"; each target maps t into its own min/max. Same proportional principle copy-paste-automation already relies on. Editor shows 0–100%.
Per-target scaling knobs on topLater, if wanted - a per-routing amount like modulators have. Not v1.

3 · Decisions & non-goals

DecisionWhy
A root-level automation track (parentless)It belongs to the tag, not to any instrument - nesting it under one member would lie about its scope. It renders as a normal automation row at root, labeled #drums · Base Color.
Targets by ParamAddress KEY, tolerant fan-outInstruments lacking the key are skipped (the lane's editor lists "drives 2 of 3 tagged"); effect-setting addresses fan out by pluginId+key match. The cluster's addressing, third consumer.
Merge position: same slot as per-instrument lanesbase ← lanes (instrument lane wins over tag lane when both exist - specific beats general, documented) ← modulators. One order, extended once.
Created from the Tracks + menu"+ ▸ Tag automation ▸ #tag ▸ param key" (keys = union across tagged instruments). The add flow needs a tag to exist first - greyed otherwise.
Non-goalsTag hierarchies · per-target offsets · tag-scoped EFFECTS (different feature) · retro-compat migration (new optional track fields only).
The one-sentence mental model: a parentless lane that says "#drums · hue: t" in normalized terms, fanned out at resolve through the same tagIndex modulators use, each instrument stretching t into its own bounds - general lanes losing to specific ones at the one documented merge point.
