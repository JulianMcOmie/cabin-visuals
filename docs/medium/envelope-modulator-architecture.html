Envelope Modulator — Architecture

Two asks converge: "midi applies an ADSR to opacity easily" (Julia) and "a generic ability for any property" (Tyler — who correctly noticed modulators only hit ports). The answer: an Envelope modulator whose notes are gates and whose target is a ParamAddress — any numeric param, opacity first among equals.

1 · The gap it fills

2 · The shape of the thing

PieceDesign
The trackA modulator track like Pulse: its blocks/notes are the GATES. Note on = attack starts; note length = sustain hold; note off = release. Velocity scales peak. Pure function of beat: at any beat, the envelope value is computable from the gate notes alone (closed-form per segment — no integrator state, scrub-exact).
Paramsattack · decay · sustain (level 0–1) · release — all in BEATS (tempo-musical, deterministic), plus depth (−1..1).
TargetA ParamAddress (the effect-param-automation cluster type) + the existing scope (track/tag/subtree) + blend: multiply (default — opacity, scale) or add (offsets). Ports remain valid targets too — Envelope with a port target is just a fancier Pulse.
Merge orderbase param ← automation lane ← modulator blend. Automation states intent over time; the envelope articulates on top of it. One documented order, applied in computeAtBeat.
Opacity itselfA UNIVERSAL param, not per-instrument: opacity joins the object state and ObjectRenderer applies it at the placement group (traversing cached materials, transparent switch at <1). Every instrument becomes fade-able without touching any instrument file — the feature's whole point.

3 · Decisions & non-goals

DecisionWhy
Depends on the ParamAddress clusterTargeting "any param" IS the richer-target design; building a private targeting scheme here would fork it — the exact mess the cluster exists to prevent.
Beats, not seconds, for ADSRDeterminism (pause invariant) and musicality (a 1-beat release stays musical when bpm changes); it's also what the engine can evaluate closed-form at any beat.
Renderer-level opacityPer-instrument opacity params would mean 28 edits and inconsistent semantics; one group-level application is uniform, cheap (cached material list per object), and canary-safe (function of beat).
Multiply as default blendEnvelope 0 = param 0 (invisible/collapsed), envelope 1 = untouched — the intuitive articulation semantics for opacity/scale/radius.
Non-goalsCurved ADSR segments (linear first; curves inherit per-keyframe work later) · LFO modulator (obvious sibling — name it, don't build) · audio-reactive sources · retrigger polyphony rules beyond "latest gate wins, releases overlap-max".
The one-sentence mental model: a Pulse that grew a real ADSR and learned the cluster's param addressing — its notes gate a closed-form envelope that multiplies into any param, with a renderer-level opacity param created so "fade anything with MIDI" is the out-of-box demo.
