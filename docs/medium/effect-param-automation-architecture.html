Effect-Param Automation — Architecture

Automation lanes learn a second kind of target: an effect instance's setting. The real deliverable is the param address — one addressing type that this feature, the envelope modulator, and tag automation all share. Design it once here; the other two become consumers.

1 · The param address — the cluster's shared decision

// The one new concept. Everything that "aims a value at a knob" uses it.
type ParamAddress =
  | { kind: 'param';  key: string }                          // instrument param (today's targetParam)
  | { kind: 'effect'; instanceId: string; key: string }      // an EffectInstance's setting   ← this doc
// (the envelope modulator aims a ParamAddress; tag automation fans a ParamAddress
//  across tagged tracks — same address, different scopes)

2 · What the user sees

SurfaceBehavior
Add-automation menu (track label right-click)Grows one group per effect instance: "Radial Symmetry ▸ radius · count", after the instrument params. One-lane-per-address rule carries over.
The laneIdentical editing to param lanes — value rows scaled to the setting's min/max, same interpolation modes, same pitch-encoded keyframes.
Effect deletedIts lanes enter the param-not-found state (keyframes preserved, re-aimable) — same treatment as a renamed instrument param.
Symmetric effectsThe payoff item: radial copy's radius under a lane is the demo this feature exists for — that pair sequences immediately after this one.

3 · Decisions & non-goals

DecisionWhy
targetEffect beside targetParam, not replacing itAdditive optional fields → no document upgrade; old lanes untouched. (A later migration to a single ParamAddress field can ride the NEXT schema bump, whenever one happens for other reasons.)
Address by instanceId, never by plugin id or list indexTwo echoes on one track must automate independently; reordering effects must not re-aim lanes.
Overlay merges in computeAtBeat, wrappers read mergedThe params overlay's exact twin — one mental model, one code shape, canary-safe.
Numeric settings onlySame rule as params (keyframes interpolate numbers). Enabled/bypass automation is NOT this feature.
Non-goalsModulator→effect routing (comes free later via ParamAddress if wanted) · per-keyframe curves (orthogonal) · automating enabled.
The one-sentence mental model: mint the ParamAddress type, give effect settings the same per-frame overlay instrument params already have, point the wrappers at the merged values — and two future features (envelope modulator, tag automation) inherit their targeting for free.
