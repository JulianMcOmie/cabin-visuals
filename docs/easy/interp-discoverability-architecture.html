Interp Discoverability — Architecture

Per-lane animation curves exist and work — Julia looked for them and concluded they didn't. The feature is fine; the surface is the bug. Three candidate treatments, one hiding-defect fix, and a decision Julia has deliberately left open (★).

1 · Why a working feature reads as missing

2 · The candidate treatments (★ Julia decides)

OptionShapeTrade
1 · Promote in placeRename to "Curve", give it a label + width, separate it from the grid/zoom cluster with its own toolbar group when an automation lane is open.Cheapest; still only visible once you're inside the piano roll.
2 · Timeline right-clickAdd "Curve ▸ step/linear/…" to the automation lane's context menu (the existing TrackContextMenu opens on the parent — this needs a small menu on the automation row itself).Discoverable without opening the editor; a second surface to keep in sync (it's one store field, so sync is free).
3 · Lane-label chipThe automation row's label shows the current mode as a tiny chip ("lin", "exp"); clicking cycles or opens a mini-menu.Always visible — strongest discoverability; adds chrome to every automation row.

These compose: 1 is worth doing under any outcome; 2 and 3 are additive. The recommendation below assumes 1+2.

3 · Decisions & non-goals

DecisionWhy
Fix the silent-hide unconditionallyWhen the parent-param lookup fails (param renamed/instrument swapped), show the control anyway with generic bounds — a lane whose curve you can't see or change is strictly worse than one with unlabeled units.
Recommend 1 + 2, hold for ★Promote-in-place is free; the right-click entry is the DAW-native discovery path ("right-click shows me what this row can do"). Option 3 waits until automation rows grow other chrome (per-keyframe curves will want row UI too — design them together then).
Per-lane onlyPer-keyframe curves are their own medium item; this task must not grow into it.
The one-sentence mental model: the curves shipped, the signpost didn't — fix the vanishing-control defect for certain, then pick how loud the signpost should be (toolbar promotion, right-click entry, or a label chip), which is exactly the ★.
