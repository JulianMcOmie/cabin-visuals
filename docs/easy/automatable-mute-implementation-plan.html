Automatable Mute — Implementation Plan

Verify what the Mute modifier lane actually does to a persistent object; then either write two sentences of documentation or wire the lane into the renderer's existing hide-switch.

B · Phase-by-phase

Phase 0 — Verify on both instrument shapes (~10 minutes, no code)

  • Track A: a note-driven instrument (Particle Burst-like). Add a Mute modifier lane, draw a region over bars 2–4, play through. Expected: flashes stop inside the region.
  • Track B: a persistent-body instrument (Cube). Same lane, same region. The question: does the cube body disappear during the region, or only its note responses?
  • Also confirm scrubbing INTO the region shows the muted state instantly (pure-function-of-beat check).

Phase 1a — If the body already hides: document, done

  • One line in the README's editor tour ("Mute modifier = timed mute, M button = static mute") and a message to Tyler. Close the item.

Phase 1b — If the body persists: wire the lane to the hide-switch

  • edit core/visual/resolve.ts: where the parent's modifier lanes are gathered, emit the Mute lane's regions as beat-ranges on the resolved object (muteRegions: {start, end}[]) — data, not per-frame work.
  • edit core/visual/VisualEngine.ts (computeAtBeat): object hidden = track.muted || inAnyMuteRegion(beat) — same flag the M-button path sets on the object state.
  • edit components/visual/ObjectRenderer.tsx: no change if it already reads the single hidden flag — the point of routing both mutes through one switch.
  • Region edges are hard cuts on the beat — deterministic, scrub-exact, export-exact.

Phase 2 — Verify

  • Cube + Mute region bars 2–4: visible → gone at bar 2 exactly → back at bar 4 exactly, identical under play, scrub, and a 4-bar export.
  • M button and Mute lane compose: M-muted track stays hidden regardless of lane regions.
  • Canary green while parked inside and outside the region.

  • risk Semantic surprise: someone may WANT "mute events but keep the body." If so, that argues for keeping Mute = events-only and adding a separate Hide modifier type instead — surface this to Julia/Tyler before choosing 1b's exact semantics.
  • note Trigger-style instruments already behave as "gone" under the event mute — the verification's track A vs track B split is what makes the decision informed.
  • note Fades stay out: one fade system (envelope modulator) later, not a second one here.
