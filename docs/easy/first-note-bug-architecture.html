First-Note Bug — Architecture

Tyler: "pressing play from the start doesn't play the first note — maybe it starts from the first note already being played, instead of right before it." An investigation design: three hypotheses, one decisive experiment each, and the fix that follows from whichever survives.

1 · The hypothesis map

The report is ambiguous between missing (the note never renders) and pre-fired (the note is already at its peak the instant play starts, so its attack is never seen). The system's shape makes the second far more likely: instrument envelopes are decay-from-onset — at exactly beat == note.beat the envelope is at MAXIMUM, and it only falls from there. A playhead parked at beat 0 with a note at beat 0 is parked on the peak.

3 · The fix, per outcome

If it's…Then the fix is…
H1 · peak-at-zeroA design choice, not a patch: either a short attack ramp (2–4 frames beat-derived, stays a pure function of beat) on note envelopes, or a small transport pre-roll option (play starts a quarter-beat early), or simply "put your first note at beat 0.25". Decide with Tyler — the doc's recommendation is the attack ramp, applied in the shared envelope helpers so every instrument gets it at once.
H2 · first-frame skipThe transport emits one frame at exactly the start beat before the RAF clock takes over — a one-line guarantee in core/playback.ts.
H3 · comparator driftNormalize every onset test to beat >= n.beat; add the convention to the instrument-authoring notes.
The one-sentence mental model: the pause invariant makes this cheap to diagnose — park the playhead where play "skips," and whatever the frozen frame shows IS what playback computes there; if the note already looks fired while parked, nothing is skipped and the conversation is about attack ramps, not bugs.

Investigation architecture — the step-by-step lives in first-note-bug-implementation-plan.html. Companion facts: envelopes are pure decay-from-onset by design (pause-invariant push); the transport anchors (beat, when) at play and the RAF tracker advances from there.
