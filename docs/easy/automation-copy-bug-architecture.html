Automation Copy Bug — Architecture

Tyler: "automating a parameter seems to change the parameter on a copy of the object instead of the object itself." Three readings of that sentence, ordered by likelihood, and the experiment that separates them.

1 · How automation actually flows — and where a "copy" could appear

Automation lanes never write to the document's params. They're sampled per frame inside the engine and overlaid on top of the base params before the instrument sees them — the slider's stored value is deliberately untouched. Two places in that pipeline can read as "a copy changed":

2 · The discriminating experiment

ONE cube, moved; slider unchangedReading A — engine is correct, the TrackEditor needs live-value feedback. The fix is UI: show the automated value (live readout or a second indicator on the slider) and badge the param as automated.
TWO cubes (source + displaced twin), no effects on the trackReal duplication bug — inspect the resolved graph and the renderer's group tree; something mounts the object twice.

3 · Decisions & non-goals

DecisionWhy
The overlay design staysAutomation NOT writing the document is load-bearing: undo, autosave, and the slider's role as "the base value" all depend on it. Whatever the outcome, the fix is presentation or wrapper order — never "write automation into params".
If Reading A: show, don't mergeThe slider keeps showing the base (it's editable); a separate live-value affordance (thin marker on the slider track + an "automated" badge that links to the lane) shows what the engine computed this frame.
Repro against Tyler's project firstHis exact wording ("a copy of the object") smells like clone effects were involved — don't dismiss Reading B without his setup.
The one-sentence mental model: automation is an engine-side overlay that never touches the document — so either the panel is failing to show the overlay (add live feedback), or an effect wrapper is composing the overlay in the wrong order (fix the chain); park the playhead and the frozen frame tells you which.
