B · Phase-by-phase

Phase 0 — Reproduce requires Tyler's project or params

  • Get the exact setup: which instrument, note at which beat, bpm. Failing that: Cube + one note at beat 0, 120bpm — the minimal candidate.
  • Record what "doesn't play" means on screen: never appears vs appears-mid-decay vs appears without attack.

Phase 1 — Discriminate (no code changes, ~15 minutes)

  • H1 check: park the playhead at beat 0 (SkipBack). If the note's visual is already at full intensity while paused, nothing is being skipped — the envelope peaks at onset by design. (The pause invariant guarantees this parked frame ≡ playback's frame at 0.)
  • H2 check: temporary log in core/playback.ts's RAF tick — the first 10 beats after play(). Starts at 0.000 → H2 dead. Starts at ~one-frame-worth → H2 confirmed.
  • H3 check: grep -n "beat >" src/editor/instruments/ vs "beat >=" — comparator outliers on onset tests.

Phase 2 — The fix (one of three)

  • H1 Add a short beat-derived attack to the shared envelope shape: env = attack(t) · decay(t) with attack(t) = min(1, t / ATTACK_BEATS), ATTACK_BEATS ≈ 0.05. Applied where envelopes are computed (per-instrument helpers; start with the instruments Tyler names). Still a pure function of beat — the canary stays green. Decision with Julia/Tyler first: ramp vs pre-roll vs "author notes off zero".
  • H2 In core/playback.ts: on play, synchronously publish the start beat once (onBeatChange(startBeat)) before the first RAF tick, so frame one is exactly the pressed-play beat.
  • H3 Normalize onset comparators to >= in the offending instruments; one sweep, one convention comment in instruments/types.ts.

Phase 3 — Verify

  • Play from 0 with a note at 0: the note's attack is visible (H1) / the first frame equals beat 0 in the log (H2) / the outlier instruments now fire at 0 (H3).
  • Scrub-land on beat 0 still shows the documented peak (H1's ramp must not break the scrub-lands-on-peak behavior for beats > 0 — the ramp only softens the first ~0.05 beat).
  • Export a 2-bar project with a beat-0 note — the first frames of the file show the same thing the editor shows (export shares the path, so this is free regression coverage).
